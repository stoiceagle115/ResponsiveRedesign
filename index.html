Responsive Redesign

Examining what makes a website function well

Process

This project was made as a way to learn about designing and prototyping websites.
The redesign process was fairly straightforward and consists of three major steps:

  1. Original Site
    A site that is suitable for a redesign is found. Afterwards, the issues of the site are identified based on four crietria: Usability, Memorability, Learnability and Acessibility.
  2. Prototyping
    • Lo-Fidelity Wireframing: using Balsamiq, quick prototypes are made in order to brainstorm possible layouts and solutions to the issues found in Part 1.
    • Visual Design Guide: using Figma, all elements necessary to make the redesign webpage are created (buttons, text style, headings, etc.)
    • Hi-Fidelity Prototypes: again with Figma, prototypes that closely resemble the final product are made.
    For all of the prototypes, a focus is given to usability. Additionally, multiple layouts are made for desktop, mobile and tablets.
  3. Redesign
    An actual, interactible website is built based on the Hi-Fidelity Prototypes. For the scope of this project, the website is limited to one page, enough to get an idea of how the site would function.

The Original Site: Rice University's Galileo Project

Interact with the site ⤵

Due to its age and the barebones structure, I felt this website would be the perfect candidate for a redesign. The website seems to have not been updated for at least a decade (the last reference of a date is around 2005), and it is only really comprised of links and text; with the occasional picture, table or slideshow.

Visit the Site

The Problems

  • The site does not stretch to fit the size of the window
    • Visit the webpage and go full-screen on a desktop or tablet to see this effect; there is large amount of void space
  • The site does not support different sized screens
    • On mobile, text is so small that it is almost unreadable; requiring the user to manually zoom in
  • The sidebar and titlecard/header are not fixed in anyway
    • For an example, click on the "Science" tab and scroll down
Site as viewed on a mobile device (375x667)

  • There are no symbols or icons to differentiate different functions
  • Every page consists only of links and text and look identical, making it difficult to navigate and remember where things are located
  • There is no search bar, the user must go to a separate page to search

To test the the Acessibility of this site, I used a program called WAVE Web Accesibilty Tool.

The main findings concerned issues with low contrast in text and the lack of a proper heading.

I do agree with the problems detected by WAVE. The website uses very light blue and gray text on a white background, which leads to very low contrast that can be very difficult to read at times. Additionaly, the website does have a titlecard but lacks a proper functioning heading.
